# Contribution Guidelines

## Overview

Digital Zimbabwe Gold (dZWG) is a Zimbabwe Gold (ZiG; ISO code: ZWG)-denominated digital settlement system designed with a strong emphasis on regulatory alignment, security, and controlled governance.

Due to the nature of the system as financial infrastructure, this repository operates under a **restricted contribution model**.

All smart contract code, protocol design, economic modeling, and core documentation are developed, reviewed, and maintained exclusively by the authorized core team to ensure:

- consistency of system architecture  
- compliance with applicable financial regulations  
- integrity of reserve-linked issuance mechanisms  
- security of on-chain and off-chain components  

The dZWG system is created and led by:

- **Simon Kapenda** — Creator of dZWG, Namibia Digital Dollar (NADD), Angola Digital Kwanza (AOAk), Naira Index (NXY), Namibia 30 Index (NAX30), Kenya 30 Index (KEN30), SACE Index (SACE), and CillarCoin  
- **Abba Payments** — Responsible for development, maintenance, and operational support  

---

## Contribution Policy

This repository **does not accept direct code contributions** from the public.

This restriction applies to:

- smart contracts  
- protocol logic  
- compliance and governance mechanisms  
- token issuance and redemption logic  
- system-level architecture  

All changes to these components are internally managed under controlled development, audit, and governance processes.

This approach is necessary to maintain institutional integrity and to ensure that no unauthorized modifications compromise system behavior or regulatory positioning.

---

## Permitted Contributions

While direct code contributions are restricted, external input is permitted in specific non-critical areas.

Contributors may:

- report documentation errors, inconsistencies, or omissions  
- suggest clarifications to improve technical or economic explanations  
- provide feedback on usability, accessibility, or adoption considerations  
- share relevant research, regulatory developments, or market insights that may strengthen the system  

All such contributions are subject to internal review before any incorporation.

---

## Prohibited Actions

The following actions are explicitly not permitted within the scope of this repository:

- submission of pull requests that modify smart contracts or core protocol code  
- alteration of compliance logic, governance structures, or contract parameters  
- unauthorized deployment of dZWG contracts intended for live or production use  
- creation of derivative systems that misrepresent dZWG as an official implementation  

Any attempt to introduce unauthorized changes to core system components will be rejected.

---

## Responsible Communication and Disclosure

All feedback, suggestions, or documentation-related issues should be submitted through GitHub Issues or designated official communication channels.

For matters related to security, vulnerabilities, or system integrity, direct communication is required.

Security-related disclosures must be submitted to:

**security@abbaii.com**

Responsible disclosure practices are expected. Public disclosure of vulnerabilities prior to coordinated review is strongly discouraged.

---

## Governance and Integrity

The restricted contribution model reflects the positioning of dZWG as a regulated, reserve-backed financial infrastructure system.

Maintaining centralized control over core components ensures:

- compliance with regulatory expectations  
- protection of users and stakeholders  
- consistency of monetary and settlement behavior  
- auditability and accountability of system changes  

External engagement remains valuable, but it must occur within a structured and controlled framework.

---

## Notice

This repository is not an open-source governance system for protocol-level modification.

It is a controlled publication layer for a financial system that requires strict oversight, security discipline, and regulatory alignment.

All official implementations, deployments, and updates of dZWG are executed exclusively under authorized governance.
